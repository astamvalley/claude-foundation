# crb-frame

cast 커맨드의 Frame 단계에서만 사용한다.

## Round 2 — 교차검증 (병렬)

Explore 3개 결과가 도착한 즉시, 3개 에이전트를 다시 **동시에** 실행한다.  
각 에이전트는 **나머지 두 Explore 결과**를 읽고 보완점을 식별한다:

| 에이전트 | 입력 | 검토 목표 |
|---------|------|----------|
| Agent A | B의 결과 + C의 결과 | A가 이미 다룬 렌즈 관점에서 B·C가 놓친 것, B+C 조합 시 보이는 새 인사이트 |
| Agent B | A의 결과 + C의 결과 | B 렌즈 관점에서 A·C가 놓친 것, A+C 조합 시 보이는 새 인사이트 |
| Agent C | A의 결과 + B의 결과 | C 렌즈 관점에서 A·B가 놓친 것, A+B 조합 시 보이는 새 인사이트 |

교차검증 프롬프트 구조:
```
다음은 다른 두 Explore 에이전트의 분석이다:

[Explore X]
...

[Explore Y]
...

이 두 분석을 함께 읽으면서:
1. 두 에이전트가 모두 놓친 <내 렌즈> 관점의 중요한 질문이나 리스크가 있는가?
2. 두 분석을 조합했을 때 단독으로는 보이지 않던 새로운 인사이트가 있는가?
3. 두 에이전트의 발견 중 방향 수정이 필요한 항목이 있는가?

발견한 것만 간결하게 출력하라. 없으면 "보완 없음"으로 종료.
```

## 합의 수렴

Round 1 + Round 2 전체 결과를 취합해 아래를 식별한다:

**합의 영역** (3개 모두 동의)
→ 다음 단계의 기반이 되는 확실한 전제

**부분 합의** (2/3 동의)
→ 가능성 높은 방향, 단 검증 필요

**교차 발견** (Round 2에서만 드러난 것)
→ 단독 분석으로는 보이지 않던 인사이트 — 별도 강조

**긴장점** (의견 충돌 또는 1개만 제기)
→ 명시적으로 결정이 필요한 지점

합의가 없어도 진행한다 — 긴장점 자체가 Design의 재료다.

## 사용자 확인 (--auto 없을 때)

Frame 요약을 출력하고 확인을 요청한다:

```
─── Frame 완료 ───────────────────────────
[합의/부분합의/긴장점 요약]
─────────────────────────────────────────
이 방향으로 Design 단계를 진행할까요?
계속하려면 Enter, 수정이 필요하면 피드백을 입력하세요:
```

**--auto 있을 때**: 요약 출력 후 자동으로 Design 진행.

## 피드백 루프

사용자 피드백이 있으면 Frame을 수정하고 재확인한다. 반복 횟수를 누적한다.

**2회 이상 반복 시** 아래 선택지를 제시한다:

```
─── Frame 수정 완료 (N회째) ─────────────
[수정된 요약]
─────────────────────────────────────────
계속 수정하거나, 현재 방향으로 Design을 진행할 수 있습니다.
> Design 진행
> Frame 다시 수정
```

## 실행 데이터 보관

Frame 진행 중 아래 데이터를 컨텍스트에 누적한다. run-log.jsonl 기록은 커맨드 완료 시 `crb-output` 스킬이 처리한다.

```
phases.frame.consensus = [...]
phases.frame.tensions = [...]
phases.frame.user_feedback = []   ← 피드백마다 추가
phases.frame.iterations = 0       ← 반복마다 증가
```

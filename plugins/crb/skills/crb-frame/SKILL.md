# crb-frame

cast 커맨드의 Frame 단계에서만 사용한다.

## 합의 수렴

3개 Explore 결과를 취합해 아래를 식별한다:

**합의 영역** (3개 모두 동의)
→ 다음 단계의 기반이 되는 확실한 전제

**부분 합의** (2/3 동의)
→ 가능성 높은 방향, 단 검증 필요

**긴장점** (의견 충돌 또는 1개만 제기)
→ 명시적으로 결정이 필요한 지점

합의가 없어도 진행한다 — 긴장점 자체가 Design의 재료다.

## 사용자 확인 (--auto 없을 때)

Frame 요약을 출력하고 확인을 요청한다:

```
─── Frame 완료 ───────────────────────────
[합의/부분합의/긴장점 요약]
─────────────────────────────────────────
이 방향으로 Design 단계를 진행할까요?
계속하려면 Enter, 수정이 필요하면 피드백을 입력하세요:
```

**--auto 있을 때**: 요약 출력 후 자동으로 Design 진행.

## 피드백 루프

사용자 피드백이 있으면 Frame을 수정하고 재확인한다. 반복 횟수를 누적한다.

**2회 이상 반복 시** 아래 선택지를 제시한다:

```
─── Frame 수정 완료 (N회째) ─────────────
[수정된 요약]
─────────────────────────────────────────
계속 수정하거나, 현재 방향으로 Design을 진행할 수 있습니다.
> Design 진행
> Frame 다시 수정
```

## 상태 기록

Frame 완료(또는 각 반복) 후 state.json의 `phases.frame` 필드를 업데이트한다:

```json
"frame": {
  "consensus": ["합의 항목1", "합의 항목2"],
  "tensions": ["긴장점1", "긴장점2"],
  "user_feedback": ["피드백1", "피드백2"],
  "iterations": 1
}
```

피드백이 있을 때마다 `user_feedback` 배열에 추가하고 `iterations`를 증가시킨다.
